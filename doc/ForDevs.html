For Developers

In order to better understand the internals of how ADI's transceiver and high-speed Converter Toolbox system objects are implemented, it is also useful to understand the overall structure of MATLAB's libiio package and how ADI's common interface layer serves as the shim layer on which both Transceiver Toolbox and High-Speed Converter Toolbox rely.

matlabshared.libiio

First, let us take a look at the list of superclasses that the base class in MATLAB's libiio package inherits from.
superclasses('matlabshared.libiio.base')
Superclasses for class matlabshared.libiio.base: matlab.System handle matlab.mixin.CustomDisplay matlab.system.mixin.Propagates matlabshared.libiio.context matlabshared.libiio.channel matlabshared.libiio.device matlabshared.libiio.buffer matlabshared.libiio.top
As shown by the output displayed, the matlabshared.libiio.base is a child class of the remaining classes that get installed when MATLAB's Communications Support Toolbox for Xilinx Zynq Based Radios. Additionally, since this class is the parent class for the classes in the common interface layer, we can also look at the abstract properties and methods that the child classes of matlabshared.libiio.base are expected to implement.
meta.abstractDetails('matlabshared.libiio.base')
Abstract methods for class matlabshared.libiio.base: releaseChanBuffers % defined in matlabshared.libiio.base configureChanBuffers % defined in matlabshared.libiio.base Abstract properties for class matlabshared.libiio.base: SamplesPerFrame % defined in matlabshared.libiio.base devName % defined in matlabshared.libiio.base kernelBuffersCount % defined in matlabshared.libiio.base dataTypeStr % defined in matlabshared.libiio.base channelCount % defined in matlabshared.libiio.base DataTimeout % defined in matlabshared.libiio.base isOutput % defined in matlabshared.libiio.base
When we look at the properties list for adi.common.RxTx class,
properties('adi.common.RxTx')
Properties for class adi.common.RxTx: EnabledChannels uri enIO SamplesPerFrame devName kernelBuffersCount dataTypeStr channelCount DataTimeout
we can notice that all the properties have been implemented by this class. Note however, that this class is still an abstract class which defines certain other properties which its child classes must implement. So, it cannot be instantiated.
c = adi.common.RxTx
